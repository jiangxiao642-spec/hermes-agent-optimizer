502 错误修复指南（跟着做就行）
══════════════════════════════════════════

你的问题：路由中间件找不到 DeepSeek 的 API key，所以报 502。
解决：创建一个 .env 文件，把 key 写进去。

════════════════════
你需要准备：
════════════════════

1. DeepSeek API key
   如果没有 → 去 https://platform.deepseek.com 注册账号
   注册后在「API Keys」页面创建一个 key，格式是 sk- 开头的一串字母数字

════════════════════
操作步骤（3步）：
════════════════════

【第1步】找到你下载的 hermes-agent-optimizer 文件夹
  比如你解压到了 D:\hermes-agent-optimizer

【第2步】在文件夹里创建 .env 文件
  打开记事本 → 粘贴下面这行（把 *** 替换成你的 key）：

    DEEPSEEK_API_KEY=sk-你的key

  然后点「文件 → 另存为」：
    - 保存位置：选 hermes-agent-optimizer 文件夹
    - 文件名输入：.env  （注意：开头有个点）
    - 保存类型选：所有文件(*.*)
    - 编码选：UTF-8
    - 点保存

  ★ 如果提示"必须输入文件名"，用这个办法：
    键盘按 Win+R → 输入 cmd → 回车
    在黑色窗口里输入（替换成你的实际路径和key）：

    echo DEEPSEEK_API_KEY=***> D:\hermes-agent-optimizer\.env

    回车，窗口关掉。

【第3步】重新跑
  关掉之前报错的 PowerShell 窗口
  重新双击启动脚本，或者在 PowerShell 里重新输：

    python route_and_prepare.py

  这次应该正常启动，不会报 502 了。

════════════════════
验证是否成功：
════════════════════

  看到这个就对了：
    知识路由中间件启动 → http://0.0.0.0:8000
    路由表 6 个规则
    后端 → https://api.deepseek.com/v1/chat/completions

  如果还是报错，把错误信息截图发过来。
