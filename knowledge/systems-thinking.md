# 系统思维

## 核心概念

| 概念 | 一句话 | 应用 |
|------|--------|------|
| 存量 | 系统里累积的东西 | GitHub star数、skill数、用户数 |
| 流量 | 改变存量的速率 | 每天新增star、流失用户 |
| 增强回路 | 自我强化，越好越好 | 更多用户→更多反馈→更好→更多用户 |
| 调节回路 | 自动刹车，保持稳定 | Token成本上升→被迫减少调用→增速放缓 |
| 延迟 | 因果之间的时间差 | 做好→有人star→有人付费，中间有时间 |

## 十二杠杆点（从弱到强）

干预系统时，起点越低越省力但效果越弱，越高越费力但效果越强。

| 层 | 名称 | 例子 |
|----|------|------|
| 12 | 调参数 | 换API provider、调整定价 |
| 11 | 调缓冲 | 加大token预算、加内存 |
| 10 | 调物理结构 | 加服务器、扩带宽 |
| 9 | 调延迟 | 缩短用户反馈周期 |
| 8 | 调负反馈 | 加熔断器、限流 |
| 7 | 调正反馈 | 减少增强回路阻力 |
| 6 | 调信息流 | 更好的监控、日志 |
| 5 | 调规则 | 激励结构（bounty定价） |
| 4 | 调自组织 | Agent自主决策权 |
| 3 | 调目标 | 从"更多功能"到"更好体验" |
| 2 | 调心智模型 | 从"工具"到"助手"的身份切换 |
| 1 | 超越范式 | 换赛道 |

## 系统衰退模式

- **公地悲剧：** 共享资源被过度消耗（如免费API被刷爆）
- **上瘾回路：** 短期奖励掩盖长期伤害（过度依赖快捷方式）
- **富者愈富：** 头部Agent吃更多资源，新agent饿死
