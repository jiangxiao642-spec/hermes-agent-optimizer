# 协作模式

## 多 Agent 协作原则

1. **分工信号：** 编程 Agent 适合"快进快出"的活（改一段代码、写一个脚本），协调 Agent 适合"长周期"的活（项目管理、用户跟进、技能积累）。

2. **不互相替代：** 各做各擅长的。编程 Agent 不会变成管家，管家也不会变成程序员。

3. **信息不对等是优势不是问题：** 持久记忆是协调 Agent 的壁垒，快速执行是编程 Agent 的壁垒。

## 通信架构

```
用户
  ↓
协调 Agent（持久记忆 + 路由）
  ├─→ 编程 Agent（Claude Code / Codex）
  ├─→ 视觉 Agent（桌面控制）
  ├─→ 记忆 Agent（agentmemory）
  └─→ 知识 Agent（Context7）
```

## 委托规则

| 任务类型 | 委托给 | 原因 |
|----------|--------|------|
| 代码编写/重构 | 编程 Agent | 专注、不废话、有工具链 |
| 视觉/桌面操作 | 桌面 Agent | 有截图/点击/UIA 权限 |
| 记忆/知识 | 记忆 Agent | 有长期存储和检索 |
| 任务拆解/路由 | 协调 Agent | 有全局视图 |

## 通信协议

1. **Mailbox 模式：** 写文件 → 对方读取 → 回复写文件。适合非实时通信。
2. **Signal 模式：** agentmemory signal 系统，实时消息。适合高频率交互。
3. **API 直连：** 对方暴露 HTTP API，直接调用。适合同主机上的 Agent。

## 权限边界

| 操作 | 协调 Agent | 编程 Agent | 桌面 Agent |
|------|-----------|-----------|-----------|
| 读取文件 | ✅ | ✅ | ❌ |
| 写入文件 | ✅ | ✅ | ❌ |
| 执行命令 | ✅ | ✅ | ✅（仅GUI） |
| 修改配置 | ✅ | ❌ | ❌ |
| 对外发布 | ❌ | ❌ | ❌ |
